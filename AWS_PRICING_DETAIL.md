## 4. Application Load Balancer（ALB）

### 公式料金（東京リージョン）
- **ALB時間**: $0.0243/時間
- **LCU**: $0.008/LCU/時間

### LCU（Load Balancer Capacity Unit）とは
以下のうち最も高い値が課金対象:
- 新規接続数: 25接続/秒
- アクティブ接続数: 3,000接続/分
- 処理バイト数: 1 GB/時間
- ルール評価数: 1,000評価/秒

### 構成（小規模想定）
- **新規接続**: 5接続/秒 = 0.2 LCU
- **アクティブ接続**: 500接続/分 = 0.17 LCU
- **処理バイト**: 100 MB/時間 = 0.1 LCU
- **ルール評価**: 50評価/秒 = 0.05 LCU

**平均LCU**: 0.2 LCU（最大値を採用）

### 計算
```
ALB時間料金:
$0.0243 × 730時間 = $17.74

LCU料金:
0.2 LCU × 730時間 × $0.008 = $1.17

合計: $18.91 ≈ 2,875円
```

**根拠URL**: https://aws.amazon.com/jp/elasticloadbalancing/pricing/

## 重要な注意点

### 1. 無料利用枠（Free Tier）
新規アカウントの場合、12ヶ月間は以下が無料:
- **EC2/Fargate**: 一部無料枠あり
- **RDS**: db.t2.micro 750時間/月
- **S3**: 5 GB ストレージ
- **CloudFront**: 1 TB データ転送
- **CloudWatch**: 10メトリクス

**→ 初年度は実質コストが50-70%削減される可能性あり**

### 2. リザーブドインスタンス（RI）
- **1年契約**: 約30-40%割引
- **3年契約**: 約50-60%割引

**→ 継続利用が確定している場合、年間5万円以上削減可能**

### 3. Savings Plans
- **Compute Savings Plans**: 最大66%割引
- **EC2 Instance Savings Plans**: 最大72%割引

### 4. 変動要素
以下の要素で実際のコストは変動:
- **トラフィック量**: ユーザー数・アクセス数に比例
- **ストレージ増加**: 添付ファイル・ログの蓄積
- **データベースサイズ**: データ増加に伴うストレージ・IOPS増
- **機能追加**: 新サービス導入時
