## 1. 依頼の背景

弊社は現在、建設業向け施工管理SaaS「ダンドリワーク」（https://dandori-work.com/）を提供しておりますが、新たにダンドリワーク契約企業向けの**日報管理システム**を開発いたしました。

本システムは、ダンドリワークとは**別のインフラ基盤（別ドメイン・別データベース・別認証）**で運用するため、既存のダンドリワーク利用規約のオプション扱いではなく、**独立した利用規約**として整備する方針です。

一方、**プライバシーポリシー**については、弊社の既存ポリシー（https://dandori-work.co.jp/privacy/ ）が会社単位の方針であることから、新規に作成するのではなく、**既存ポリシーに日報システム固有の項目を追記する**方針で考えております。

## 2. システムの概要

本システムは、建設会社の社員が日々の業務報告を作成・提出・承認するためのWebアプリケーションです。詳細は**別紙「システム仕様書」**に記載しておりますが、主な特徴は以下のとおりです。

### 対象ユーザ
- 建設会社の役員・従業員・業務委託先の作業者（1社あたり数名〜数十名規模）

### 主な機能

| 機能 | 概要 |
|------|------|
| **営業日報** | 訪問先、担当者名、訪問時間、訪問内容、交通費等を記録 |
| **作業日報** | 物件名、天候、作業時間、作業者記録（氏名・労働時間・作業種別）、使用材料（品名・数量・単価・金額）、外注先（名称・人数・作業内容）を記録 |
| **承認ワークフロー** | 日報を提出→管理者が承認または却下。承認ステータス（未提出/提出済/承認/却下）を管理 |
| **集計・分析** | 労働時間の自動集計（昼休み12:00-13:00の自動控除、残業時間の自動計算を含む）、使用材料費・外注費の物件別集計 |
| **マスタ管理** | 物件、使用材料、外注先、工事種別、単位の基礎データを管理者が登録・管理 |
| **ユーザ管理** | 管理者が画面上で社員の追加・削除・権限設定を実施（弊社への申請不要） |
| **監査ログ** | 全操作履歴を自動記録し、管理者が閲覧可能 |
| **テンプレート** | 作業日報の雛形を保存・共有する機能 |
| **通知** | 日報提出・承認・却下時に関係者へ通知 |

### 技術構成
- Webブラウザからアクセスするクラウドサービス（SaaS）
- ダンドリワークとはドメイン、サーバ、データベース、認証基盤がすべて独立
- ダンドリワークとのデータ連携・シングルサインオン等は現時点で未実装

## 4. プライバシーポリシーの追記方針

### 現行プライバシーポリシーの概要

弊社の現行プライバシーポリシー（別紙④）は、**会社単位**の個人情報保護方針であり、以下の利用目的が定められています。

- 取引先情報：業務管理・請求・支払い管理
- 従業者情報：労務・人事・給与関連業務
- 採用応募者情報：採用判断・連絡
- 退職者情報：連絡・問合せ対応
- お問合せ情報：資料請求・セミナー申込回答
- 会員登録情報：本人認証・ログイン機能
- 受託業務情報（非開示）：建設業務向けプロジェクト管理契約遂行

現行ポリシーには、日報管理システムで取得する個人情報（作業者氏名、訪問先担当者名、労働時間記録等）及びその利用目的が**記載されておりません。**

### 追記案

既存ポリシーの「当社が取扱う個人情報」セクションに、以下の項目を追加する方針です（追記案全文は別紙⑤に記載）。

**追加する個人情報の種類：**
- 利用ユーザの氏名、ログインID
- 日報に記載される作業者の氏名、労働時間、作業内容
- 日報に記載される訪問先担当者の氏名
- 承認者の氏名、承認日時
- 操作ログ（操作者、操作日時、操作内容）

**追加する利用目的：**
- 日報管理システムのサービス提供（日報の作成・提出・承認機能の提供）
- 利用ユーザの本人認証及びアクセス制御
- サービスの利用状況の分析及び改善
- サービスに関するお知らせ及びサポートの提供
- 利用料金の請求

**追加する第三者提供・委託に関する記載：**
- 法令に基づく場合を除き、本人の同意なく第三者に提供しない旨
- サービス提供に必要な範囲でクラウドインフラ事業者等に委託する場合がある旨

## 5. 特にご確認いただきたい事項

以下の論点について、特に慎重なご確認をお願いいたします。

### 【最重要】労働関連データの法的リスク

本システムは**作業者の労働時間を記録・集計**します。これに関連して：

**(a) 労働基準法上の帳簿保存義務との関係**
- 本システムの日報データが労基法第109条の「労働関係に関する重要な書類」に該当する可能性
- 該当する場合、弊社（SaaS提供者）としてのデータ保持義務の有無・範囲
- 現行ドラフト第20条4項では「契約者自身のバックアップ責任」としていますが、この整理で十分か

**(b) 労働時間集計の自動計算に関する免責**
- 昼休み（12:00-13:00）の自動控除、残業時間の自動計算等のロジックを実装
- これらが契約者の就業規則と異なる場合の責任範囲
- 現行ドラフト第23条で免責としていますが、この整理で十分か

**(c) データ消去の適切性**
- 契約終了後30日以内のデータ消去（第31条2項）が、労基法上の保存義務期間（5年、経過措置3年）と矛盾しないか

### 【重要】承認ワークフローの法的効力

**(d) 承認記録の法的位置づけ**
- 本システム上の「承認」は社内業務上の承認であり、法的な承認行為とは異なる旨を第19条2項に記載
- この免責の書き方が適切か、また不十分な点はないか

**(e) 承認権限の濫用に関する禁止事項**
- 第9条(15)で「正当な理由なく承認を拒否する行為」を禁止しているが、SaaS提供者として契約者社内の承認運用にどこまで関与すべきか

### 【重要】契約形態の妥当性

**(f) 独立した利用規約とすることの妥当性**
- ダンドリワークのオプション機能ではなく、別契約とする方針の法的妥当性
- 第7条（ダンドリワーク契約者向け特則）の書き方が、両契約の関係を適切に整理できているか

**(g) ダンドリワーク規約との整合性**
- 両規約間で矛盾する条項がないか
- ダンドリワーク契約者が本サービスも利用する場合の、秘密保持義務・免責条項等の重複適用に問題がないか

### 【重要】プライバシーポリシー関連

**(h) 既存プライバシーポリシーへの追記で十分か**
- 日報システム専用のプライバシーポリシーを別途作成する必要はないか
- 既存ポリシーへの追記で法的に問題ないか（同一法人が運営する別サービスとして）
- 追記の場合、既存ポリシーの施行日・改定日の扱いをどうすべきか

**(i) 追記案の内容の過不足**
- 別紙⑤の追記案で、個人情報保護法第21条（利用目的の通知又は公表）の要件を満たしているか
- 取得する個人情報の種類に漏れはないか
- 利用目的の記載が十分に具体的か

**(j) 訪問先担当者名の取扱い**
- 日報に記載される訪問先担当者名は、契約者の従業員ではなく**社外の第三者の個人情報**
- 契約者が本人同意なく社外の個人名を入力するケースが想定される
- 弊社（SaaS提供者）としての法的リスクと対応策

**(k) クラウドインフラ事業者への委託**
- データベース（Turso社・米国企業）にデータを保管しているが、サーバは東京リージョン
- 個人情報保護法上の「外国にある第三者」への提供に該当するか
- 該当する場合、プライバシーポリシーに追記すべき事項

**(l) Pマーク（プライバシーマーク）との整合性**
- 弊社はPマーク取得済みだが、新サービスの追加にあたりPマーク審査機関への届出・申請が必要か
- 追記内容がPマークの運用基準に適合しているか

### 【確認事項】その他

**(m) 入力データの知的財産権帰属**
- 利用規約第29条2項で入力データの知財を契約者帰属としたが、弊社がサービス改善目的でデータを利用する際の法的整理は十分か

**(n) 損害賠償上限の妥当性**
- 現行は「直接損害に限り、過去6ヶ月分の利用料金が上限」（第32条2項）
- 日報データの喪失が労基法違反につながる可能性を考慮した場合、この上限設定は適切か

**(o) 料金体系の法的リスク**
- 月額基本費用の減額不可（契約更新時以外）の条項は消費者契約法等に照らして問題ないか
- ※本サービスはBtoB（事業者間契約）を想定していますが、念のためご確認ください

## 5. 自動計算ロジック（法的リスクに関連）

本システムには以下の自動計算ロジックが実装されています。これらが契約者の就業規則等と異なる場合、集計結果に差異が生じる可能性があります。

| ロジック | 内容 | リスク |
|---------|------|--------|
| **昼休み自動控除** | 作業時間が12:00-13:00をまたぐ場合、1時間を自動控除 | 契約者の休憩時間が12:00-13:00でない場合、計算結果が不正確になる |
| **残業時間の自動計算** | 所定労働時間を超える部分を残業として自動分類 | 契約者の所定労働時間の定義と異なる可能性 |
| **移動時間の集計** | 遠隔地への移動時間を別途記録・集計 | 移動時間の労働時間該当性は契約者の判断に委ねる |
| **時間入力の制約** | 30分刻みでの入力のみ（例：8:00, 8:30, 9:00...） | 実際の労働時間と最大29分の誤差が生じうる |

| セッション有効期間 | 30日 |
| ログイン試行制限 | 5回失敗で15分間ロックアウト |
| 通信暗号化 | HTTPS（SSL/TLS） |
| アクセス制御 | ロールベース（管理者/一般ユーザ） |
